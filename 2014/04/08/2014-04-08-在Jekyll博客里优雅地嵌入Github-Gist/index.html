    不过这同步的script载入方式存在一点问题：

    1. 如果因为众所周知的某些时不时出现的“网络原因”导致此script载入失败，之后的文章内容都会停止加载的。这不，Github今天又撞墙了，以前在博客里贴的gist都挂掉了。

    2. 即便gist会加载成功，也有可能因为速度慢而阻碍完整文章的显示速度。

    因此，我需要一种优雅地处理gist载入失败的策略。
